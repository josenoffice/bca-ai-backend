## Endpoint 2: POST /api/phase6/rewrite

### Request
```json
{
  "sectionId": 2,
  "sectionTitle": "Business Justification",
  "currentContent": "existing section text...",
  "instruction": "Make it more concise and add financial ROI figures"
}
```

### Response
```javascript
[{
  rewrittenContent: 'The rewritten section content as a string...'
}]
```

### Error Response
```javascript
[{
  status: 'error',
  phase: 6,
  errorMessage: 'Missing required fields: sectionId, instruction'
}]
```

## Endpoint 3: POST /api/phase6/download

### Request
```json
{
  "sections": [ /* array of 12 section objects (potentially user-edited) */ ],
  "metadata": { "projectTitle": "...", "businessUnit": "...", "industry": "..." }
}
```

### Response
- **Content-Type**: `application/vnd.openxmlformats-officedocument.wordprocessingml.document`
- **Content-Disposition**: `attachment; filename="ProjectTitle_BRD.docx"`
- **Body**: Binary .docx buffer

### Error Response
```javascript
[{
  status: 'error',
  phase: 6,
  errorMessage: 'No sections provided'
}]
```

## Section Source Values

| Value | Meaning | Badge Color |
|-------|---------|-------------|
| `phase5` | Extracted from Phase 5 report data | Green |
| `wizard` | User's wizard input (currentState, painPoints) | Green |
| `phase1` | Phase 1 solutions/requirements | Green |
| `phase3` | Phase 3 timeline data | Green |
| `ai` | Generated by Claude API | Blue |
| `template` | JS template (RACI, assumptions, sign-off) | Gray |
| `user` | User-edited content | Amber |

```javascript
const filename = (metadata.projectTitle || 'BRD').replace(/[^a-zA-Z0-9_-]/g, '_') + '_BRD.docx'
```
